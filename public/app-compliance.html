How It Works

The Doxa website uses two mechanisms to serve these legal pages:

1. Dynamic Routing (React Router)

The shortlinks like /delete/beacon are handled by React Router routes that redirect to the static files.

2. Static File Serving (Netlify)

All files in /public/{app-name}/ are served directly via netlify.toml routing rules.

Files Location: All legal pages are stored in the /public/{app-name}/ directories and included in the website build.

Maintenance Notes

⚠️ Critical: Do NOT delete or rename the app folders in /public/. These URLs are submitted to app stores.

When updating legal pages:

  • Keep the same filename (delete.html, privacy-policy.html)
  • Keep the same folder structure (/beacon/, /phoenix/, etc.)
  • Test all URLs after deployment
  • Update app store listings if you change the URL structure

When adding new apps:

  • Create a new folder under /public/{app-name}/
  • Add delete.html and privacy-policy.html files
  • Add routing rules to netlify.toml
  • Update this page and APP_STORE_URLS.md
  • Test before submitting to app stores
